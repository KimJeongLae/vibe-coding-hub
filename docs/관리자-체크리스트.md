# 수업 전날 관리자 체크리스트 ✅

바이브코딩 클래스 진행자(관리자)가 **수업 하루 전**에 준비할 것들.

---

## 1. 계정 초대 (하루 전날 발송 → "미리 수락해두세요" 공지)

| 서비스 | 초대 위치 | 역할 | 꼭 필요? |
|---|---|---|---|
| **GitHub** | 저장소 → Settings → Collaborators → Add people | Write | ✅ 필수 (push 하려면) |
| **Supabase** | Organization → Team → Invite member | Developer | ⚠️ SQL Editor로 테이블 만들 사람만 |
| **Vercel** | Team → Settings → Members → Invite | Member | ⛔ 대개 불필요 (자동 배포됨) |

> 💡 Supabase는 **좌석당 과금이 없어서** 인원이 늘어도 추가 요금이 없습니다. GitHub Collaborator도 무료예요.

## 2. 직원에게 공유할 정보 (수업 시작 시 Slack/문서로)

```
📎 라이브 사이트: https://vibe-coding-hub-kappa.vercel.app
📎 GitHub 저장소: https://github.com/KimJeongLae/vibe-coding-hub
📎 본인 페이지:   위 사이트에서 본인 이름 셀 클릭

🔑 Supabase 연결 정보 (.env.local 에 붙여넣기)
NEXT_PUBLIC_SUPABASE_URL=여기에-복사
NEXT_PUBLIC_SUPABASE_ANON_KEY=여기에-복사
```
> anon 키는 공개용이라 공유해도 안전합니다. (service_role 키·DB 비번은 절대 공유 금지)

## 3. 사전 점검
- [ ] 라이브 사이트가 열리고 셀이 모두 보인다
- [ ] 셀 클릭 시 각자 페이지("아직 작업물이 없어요" 안내)가 뜬다
- [ ] 직원 전원 GitHub 초대 발송 + 수락 안내
- [ ] 직원용 가이드(`docs/직원-가이드.md`) 링크 공유
- [ ] (선택) Supabase 동적 기능을 쓸 거면 `docs/supabase-연결.md` 대로 연결 + 키 공유

## 4. 공유 데이터베이스 안전 규칙 (칠판/공지)
9명이 **같은 Supabase 프로젝트**를 쓰므로:
- ✅ 새 테이블은 **본인 이름 접두사**로 (`hong_todos`, `jang_notes`)
- ⛔ 공용 `works` 테이블과 **남의 테이블 `DROP`/`ALTER` 금지**
- ⛔ SQL Editor 에서 `drop`, `truncate` 함부로 금지
