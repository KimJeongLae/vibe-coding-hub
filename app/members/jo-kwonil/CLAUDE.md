# 조권일 광고 성과 대시보드 — 작업 메모 (Claude 전용)

이 폴더(`app/members/jo-kwonil/`)에서 작업할 때 **반드시** 이 문서를 먼저 읽는다.
다른 기기·다른 세션에서 이어서 작업해도 아래 원칙과 상태를 계속 따른다.

## ⚡ 최우선 원칙: 속도 (성능이 가장 중요)
이 대시보드는 **속도가 제일 중요**하다. **클릭할 때마다 즉시(딱딱) 화면이 떠야 한다.**
새 기능을 붙이더라도 이 "즉시 반응" 원칙을 **절대 깨지 않는다.**

지켜야 할 것:
- **뷰 전환(매체 클릭 → 상세 등)은 클라이언트 상태 전환으로만.** 페이지 라우팅/전체 리로드 금지 → 클릭 즉시 렌더.
- **외부 광고 API(네이버 등)는 프리페치 + 캐시 + SWR(stale-while-revalidate).** 클릭하는 순간 캐시된 값을 즉시 보여주고, 갱신은 뒤에서. 스피너로 사용자를 기다리게 하지 말 것.
- **이미 불러온 기간(날짜 범위)은 캐시에서 즉시 표시.** 같은 조회를 반복 요청하지 않는다. (기간별 캐시 키)
- 앱 첫 로드 시 자주 볼 데이터(예: 네이버)는 **미리 불러둔다.**
- 무거운 계산은 `useMemo`, 서버 라우트는 캐시/revalidate 활용.
- 로딩은 "빈 화면 + 스피너"가 아니라 **직전 값/스켈레톤을 즉시** 보여주는 방식으로.

## 현재 구조
- `page.tsx` — 클라이언트 컴포넌트. 개요 ↔ 매체 상세 전환은 state(`selectedMedia`)로 처리(라우팅 X).
- `data.ts` — 샘플 데이터(구글/메타/카카오, 그리고 미연동 부분).
- `api/naver/route.ts` — 네이버 검색광고(SA) 실데이터. HMAC 서명 인증.
- 키는 `.env.local`(gitignore): `NAVER_SA_*`, `META_*`, `NAVER_GFA_*`.

## 연동 진행 상황 (2026-07-14 기준)
- ✅ **네이버 SA**: 실데이터 연동 완료(로컬). 상세의 SA 캠페인/상위 그룹/고성과 키워드가 실데이터.
- ⏳ **Vercel 환경변수 미등록**: 라이브에는 `NAVER_SA_*`가 없어 네이버가 샘플로 뜸. Settings→Environment Variables에 3개 등록 후 Redeploy 필요. (또는 Vercel 토큰 받아 CLI로)
- ⏳ **네이버 DA(GFA)**: OAuth 방식·파트너 승인 필요. 대행사에 Client ID/Secret/토큰/광고계정ID 요청해 둠. 받으면 `api/naver-gfa/route.ts` 만들어 SA처럼 연결.
- ⏳ **메타**: 광고계정 `378326985011472`, 액세스 토큰 발급 대기(system user 조권일, ads_read).
- ⬜ 구글/카카오: 미착수.

## 배포 규칙
- `main`에 push하면 Vercel 자동배포. **내 폴더(`app/members/jo-kwonil/**`)만** 커밋한다. 다른 사람 파일·공통 파일 건드리지 않는다.
- 수정 후 **컨펌 없이 바로 push**한다(사용자 지시). 단, push 전 로컬 렌더/타입체크는 확인.
